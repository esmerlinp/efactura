Inventario y Almacenes

Controla tus existencias, gestiona almacenes, realiza transferencias y mantén tu inventario actualizado con conteos físicos.

Gestión de Almacenes

Almacén (Warehouse)

Ubicación física donde se almacenan los productos. Cada almacén tiene un nombre, código, dirección y sucursal asociada.

Ubicación

Subdivisión dentro de un almacén: estante, rack, pasillo. Permite organizar los productos para facilitar su localización física.

¿Cómo crear y gestionar almacenes?

Accede a Inventario → Almacenes (/inventory/warehouses):

Crear un nuevo almacén:
  • Haz clic en Nuevo Almacén.
  • Ingresa el Nombre (ej. "Almacén Principal Santo Domingo").
  • Asigna un Código único (ej. "ALM-SD-001").
  • Selecciona la Sucursal a la que pertenece.
  • Ingresa la Dirección y los datos de contacto del encargado.
  • Define el Tipo de Almacén:
    • Físico: Almacén real donde se guardan productos tangibles.
    • Virtual: Almacén lógico para productos digitales o servicios.
    • Tránsito: Almacén temporal para productos en transferencia entre ubicaciones.
Gestión diaria:
  • El panel de almacenes muestra el nivel de ocupación y la cantidad de productos distintos almacenados.
  • Puedes asignar un encargado por almacén para control de responsabilidades.
Control de Stock

Entradas, salidas y ajustes de stock

El sistema registra automáticamente cada movimiento de inventario:

Entradas de stock:
  • Compras: Al registrar una factura de proveedor con productos, el sistema incrementa el stock del almacén seleccionado.
  • Notas de crédito de ventas: Cuando anulas una factura de venta, las unidades vuelven al inventario.
  • Ajustes manuales (+): Puedes registrar entradas manuales por devoluciones de clientes, sobrantes de conteo o ingresos no documentados.
Salidas de stock:
  • Facturación: Al emitir una factura de venta, las unidades se descuentan automáticamente del almacén.
  • Transferencias: La salida del almacén origen se registra como disminución de stock.
  • Ajustes manuales (-): Para mermas, roturas, vencimientos o ajustes por conteo.
Historial de movimientos:
  • Cada producto tiene un timeline (/inventory/items/{id}/movements) que muestra todos los movimientos: fecha, tipo, cantidad, documento asociado y usuario que realizó el cambio.
  • El historial es inmutable: no se puede editar ni eliminar un movimiento, solo revertirse con un movimiento compensatorio.
Alertas de Stock Mínimo

Configuración de alertas de inventario bajo

El sistema monitorea continuamente el nivel de existencias de cada producto y te notifica cuando un artículo alcanza o supera su stock mínimo:

¿Cómo funciona?
  • Cada producto tiene un campo Stock Mínimo configurable desde el catálogo (/items).
  • Cuando el stock disponible es igual o menor al stock mínimo, el sistema activa una alerta visual en el Dashboard y en el panel de inventario.
  • Los productos con alerta activa se muestran en una sección dedicada Productos por Reponer.
Recomendaciones de reposición:
  • El módulo de inventario puede sugerir cantidades de compra basadas en el historial de ventas y el lead time de reposición.
  • Puedes generar una Orden de Compra directamente desde la alerta de stock mínimo.
Si vendes un producto con stock insuficiente, el sistema te mostrará una advertencia antes de emitir la factura. Puedes optar por emitir la factura igualmente si el producto se entregará después.
Transferencias entre Almacenes

¿Cómo realizar una transferencia de stock?

Las transferencias te permiten mover existencias entre almacenes de forma controlada:

Crear una transferencia (/inventory/transfers/new):
  • Selecciona el Almacén Origen (de donde sale el producto).
  • Selecciona el Almacén Destino (a donde llega el producto).
  • Agrega los productos a transferir: busca por nombre o SKU, ingresa la cantidad.
  • El sistema verifica que haya stock suficiente en el origen.
  • Haz clic en Crear Transferencia.
Estados de una transferencia:
  • Pendiente: La transferencia ha sido creada pero aún no se ha procesado. El stock todavía está en el almacén origen.
  • En Tránsito: Has confirmado la salida del almacén origen. El stock se descuenta del origen pero aún no se ha recibido en el destino.
  • Completado: Has confirmado la recepción en el almacén destino. El stock se suma al destino y la transferencia se cierra.
Proceso paso a paso:
  • 1. Crear: Registra la transferencia con origen, destino y productos. Queda en estado Pendiente.
  • 2. Despachar: Cuando los productos salen físicamente del almacén origen, cambia el estado a En Tránsito. El stock se descuenta del origen.
  • 3. Recibir: Cuando los productos llegan al almacén destino, confirma la recepción. El sistema pasa el estado a Completado y suma el stock al destino.
Puedes transferir productos entre almacenes de la misma empresa sin implicaciones fiscales. Las transferencias no generan facturas ni movimientos contables automáticos.
Conteo Físico

Proceso de conteo físico de inventario

El conteo físico te permite verificar que las existencias registradas en el sistema coincidan con la realidad física del almacén:

1. Iniciar un conteo (/inventory/counts/new):
  • Selecciona el Almacén que deseas contar.
  • Define el Tipo de Conteo: Ciclo (grupo específico de productos) o General (todos los productos del almacén).
  • Selecciona los productos a contar (en conteo por ciclo) o deja que el sistema incluya todos (conteo general).
  • El sistema genera un Acta de Conteo con el stock actual registrado para cada producto (oculto inicialmente si es conteo ciego).
2. Registrar conteos:
  • Los usuarios designados ingresan la Cantidad Física contada para cada producto.
  • En modo Conteo Ciego, los usuarios no ven la cantidad registrada en el sistema, eliminando sesgos.
  • Una vez ingresados todos los conteos, se cierra la etapa de registro.
3. Analizar diferencias:
  • El sistema compara la cantidad contada vs. la cantidad registrada y calcula las diferencias (positivas o negativas).
  • Puedes realizar un reconteo selectivo de los productos con diferencias para confirmar.
  • Las diferencias se expresan en unidades y en valor monetario (costo unitario × diferencia).
4. Ajustes contables:
  • Una vez validada la diferencia, puedes Aprobar el Ajuste.
  • El sistema genera un movimiento contable que actualiza el stock del producto en el almacén.
  • El ajuste queda registrado en el historial de movimientos con la referencia del acta de conteo.
Los ajustes de inventario por conteo físico modifican el valor del inventario y pueden tener implicaciones contables y fiscales. Se recomienda realizarlos al cierre de cada período contable y con la aprobación del contador.
Métodos de Costeo

FIFO

First In, First Out. Las primeras unidades que entraron al inventario son las primeras en salir. El costo de venta refleja el precio de las unidades más antiguas.

Promedio Ponderado

El costo unitario se recalcula promediando el costo de todas las unidades disponibles. Usa la fórmula: (costo total del inventario) / (total de unidades).

Costo Estándar

Se define un costo fijo predeterminado para cada producto. Las variaciones entre el costo real y el estándar se registran por separado.

¿Qué método de costeo elegir?

El método de costeo determina cómo se calcula el costo de los productos vendidos (CMV) y el valor del inventario final:

FIFO (First In, First Out):
  • Ideal para productos perecederos o con fecha de vencimiento.
  • En épocas de inflación, el CMV refleja costos más antiguos (menores), generando mayor utilidad reportada.
  • El inventario final se valora a los costos más recientes.
Promedio Ponderado:
  • Suaviza las fluctuaciones de precios. Recomendado para productos con alta rotación y precios estables.
  • El costo unitario se recalcula automáticamente con cada nueva compra.
  • Es el método más común y aceptado fiscalmente en República Dominicana.
Costo Estándar:
  • Útil para industrias manufactureras con costos predecibles.
  • Requiere un proceso periódico de actualización de costos estándar.
  • Las variaciones se analizan por separado para identificar eficiencias o ineficiencias.
El método de costeo debe ser consistente con el principio contable y fiscal que aplique a tu empresa. Una vez seleccionado, solo puede cambiarse con justificación técnica y aprobación fiscal. Consulta con tu contador antes de elegir.
